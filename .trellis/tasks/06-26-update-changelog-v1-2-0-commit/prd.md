# 更新 V1.2.0 变更记录并提交

## Goal

根据当前工作区 diff，把本次已完成的功能和修复补充到 `CHANGELOG.md` 的 `V1.2.0` 条目，并按项目提交规范提交代码。

## Requirements

- 只更新 `CHANGELOG.md` 中 `V1.2.0` 的变更说明。
- 变更说明覆盖当前 diff 中的主要功能：历史统计图表重做、历史统计口径修正、文件树搜索与代码片段搜索、平台兼容和相关规约更新。
- 提交前检查 Git 状态，避免无意遗漏或误提交明显无关文件。

## Acceptance Criteria

- [ ] `CHANGELOG.md` 的 `V1.2.0` 包含本次功能点。
- [ ] 变更范围经过 Git 状态和 diff 检查。
- [ ] 代码已提交到当前分支。

## Definition of Done

- 更新发布记录。
- 执行必要的静态或版本控制检查。
- 完成一次符合近期提交风格的 Git commit。

## Out of Scope

- 不修改业务实现代码。
- 不新增依赖。
- 不推送远程分支。

## Technical Notes

- 当前工作区已有多组未提交实现变更；本任务只负责发布记录补充和提交收口。
