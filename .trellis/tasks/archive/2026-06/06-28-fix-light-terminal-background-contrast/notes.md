# Implementation Notes

- 2026-06-28: After disabling WebGL for light terminal themes, JetBrains Mono exposed horizontal glyph overlap in the default renderer. The likely root cause was global UI font CSS resetting `.xterm` descendants with `font-family: revert !important`, which can desync xterm's measured font from the user's terminal font. The fix routes xterm font CSS through `--terminal-font-family` from `XTermTerminal` and refits after renderer changes.
- 2026-06-28: Claude/Codex composer rows can leave a dark field on light themes via xterm cell attrs, not only ANSI background color. The normalizer now runs for light-theme TUI prompt rows across the visible viewport and clears explicit bg plus wide-row inverse flags while leaving isolated inverse caret cells alone.
- 2026-06-28: Codex can render the dark field on the row immediately before the visible `›` prompt. The normalizer includes one prelude row around each detected TUI prompt to clear that top strip without broadening to unrelated terminal output.
- 2026-06-28: Tab switching can re-render xterm after the visibility effect fires, which can make a stale composer strip visible again. The normalizer is now coalesced behind `requestAnimationFrame` and scheduled from xterm `onRender`, while write callbacks still run an immediate cleanup for typing responsiveness.
- 2026-06-28: Claude submitted prompt rows can move above the bottom prompt area after output arrives. The normalizer now scans the entire visible viewport instead of only the bottom rows so tab restore cannot redraw old submitted prompts with dark backgrounds.
