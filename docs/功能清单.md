## [TEMP] 增补

### Codex / Claude 历史同步

- 首次启动且尚未维护项目时，应用会延迟扫描本机 Claude / Codex 历史项目，并弹出同步选择窗口供用户确认要纳入管理的项目；已维护项目列表时启动不再自动扫描或提示。
- 同步记录会持久化保存；后续在历史会话列表点击刷新时会同时检测缺失的历史项目，也可移除已同步历史记录。
- 左侧项目列表支持展示同步来的历史分组；这些分组可打开聚合同项目历史上下文的只读终端。
- 同步历史终端会先写入聚合上下文，再发送对应 `claude` / `codex` resume 命令，避免上下文输出与启动命令抢写。
- 同步逻辑会过滤 CLI-Manager 自己生成的同步中间会话，避免重复导入。

### 终端显示与启动兜底

- 内置终端接入 Unicode 11 宽字符表，改善中文、Emoji 等宽字符在 Claude / Codex TUI 中的对齐和显示。
- 应用初始化或界面渲染失败时，会展示带错误详情的失败页，便于复制报错继续排查。
- 普通 WebView 诊断日志跟随“调试模式”输出，关闭调试模式时减少控制台噪声。

### 历史会话收藏

- 历史会话点击收藏时，会把完整会话详情快照写入本地 SQLite。
- 原始 Claude/Codex 历史 JSONL 被删除后，已收藏会话仍会在历史列表中展示，并可打开收藏时保存的只读快照。
- 取消收藏会同步删除该会话快照。
- 历史会话侧栏支持只看收藏；开启后列表和全局搜索结果都只显示收藏会话。
- 历史会话列表按 20 条分页加载，继续滚动或点击加载更多会追加下一批。
- 历史会话列表会把图片型首条消息标题显示为 `[图片]`；多张图片显示为 `[图片1][图片2]`，并补充同一条消息中的剩余文字，且会忽略图片闭合标签和重复图片前缀。
- 历史会话列表的展开、关闭和 Subagent 标识使用轻量图标样式。
- 历史会话原文中的本地图片标记会优先以内联缩略图展示，点击可放大预览，预览中按 Esc 会优先关闭图片，并保留图片标记后的正文；图片缺失或读取失败时降级显示原始标记文本。

### Git 变更面板

- Git 变更文件列表的文件右键菜单支持打开源文件，会直接在文件编辑器 Pane 中打开对应项目文件；删除状态文件不会触发打开。
- 当前活动页是项目文件编辑器时，右侧 Git 变更面板会关联到该文件编辑器所属项目。

### Git Worktree 并行隔离

- 项目支持 Worktree 隔离策略：提醒（默认）、不处理、并行时自动隔离、始终自动隔离。
- 项目已配置 CLI 工具且已有同项目终端时，新开终端可提示或自动创建独立 worktree，避免多个 AI 任务修改同一工作区；选择“不处理”时始终直接普通打开；普通未配置 CLI 的终端项目不会因启动命令或运行状态触发隔离。
- Worktree 使用项目同级 `<项目名>-worktrees/<任务名>` 目录和 `wt/<任务名>` 分支，也支持项目级自定义根目录。
- 项目树显示 worktree 子条目；终端 Tab 显示 worktree 徽标。
- Worktree 右键菜单支持打开目录、安装依赖、完成任务和丢弃。
- 完成任务向导支持提交全部改动、合并回主工作区和清理 worktree/分支；主工作区脏时阻止合并，冲突时自动中止合并。

### 文件浏览器搜索与菜单

- 文件浏览器搜索框支持在输入框内直接切换“文件 / 代码”搜索模式。
- 文件浏览器搜索框与右键菜单会跟随所在侧边面板主题配色，避免深色侧边栏中出现浅色控件割裂。
- 文件树、文件搜索结果、代码搜索结果和根空白区域右键菜单支持打开所在文件夹；文件项会在系统文件管理器中选中目标文件。
- 文件浏览器顶部项目区域支持双击打开项目位置。

## 0. V1.2.5 增补

### 0.1 终端外部文件拖拽

- 从系统文件管理器拖拽文件到内置终端时，会在终端输入位置显示本地文件路径。
- 多文件拖入时会按空格拼接多个路径。
- 路径会按当前终端 Shell 使用对应引号规则，降低空格路径或特殊字符路径输入失败的概率。

### 0.2 文件浏览器自动刷新

- 打开项目文件浏览器后，终端或外部工具修改并保存项目文件时，文件树会自动刷新，不再依赖手动点击顶部刷新按钮。
- 已展开目录会尽量保持展开状态，不因自动刷新而整棵树塌陷。
- 已打开且没有未保存修改的文件预览会自动重读最新磁盘内容。
- 已打开但仍有未保存修改的文件不会被自动覆盖。
- 文件 watcher 会携带变更路径，文件浏览器优先刷新受影响目录。
- 连续文件系统事件会串行合并后刷新，避免一次保存或批量生成文件触发多次重载和刷新乱序。
- `node_modules`、`target`、`dist`、`.gitnexus` 等高噪音目录变化不会触发文件树刷新。
- 当本机 watcher 不可用时，会自动降级为低频轮询，并在窗口重新聚焦时补一次刷新。

### 0.3 文件浏览器 Git 状态

- 文件树和文件搜索结果会标记 Git 状态，覆盖修改、新增、删除、重命名、冲突和未跟踪文件。
- 文件状态使用不同颜色区分：新增为绿色、修改为蓝色、删除为红色，未跟踪和冲突使用独立颜色。
- 目录会聚合其子文件的 Git 状态，折叠时也能看出目录内存在变更。
- 打开有 Git 变更的文本文件时，编辑器左侧标记和概览标尺会显示新增、修改、删除位置。
- 有 Git 变更的文件可通过右键菜单在文件编辑器 Tab 中打开 Diff。
- 内嵌 Diff 支持文件级、代码块级和选中行级回滚，并跟随当前终端主题背景。

### 0.4 Git 面板多子仓库监控

- 项目根目录下存在多个 git 仓库时，Git 面板顶部显示仓库切换下拉，列出主仓库与限深扫描到的子仓库（含当前分支）。
- 切换子仓库后，变更列表、分支状态、暂存、提交、推送、拉取与 Diff 均针对所选子仓库生效。
- 扫描排除 `node_modules`、`target`、`dist` 等常见大目录并跳过符号链接，避免面板首开卡顿。
- 子仓库内文件改动会触发面板自动刷新；子仓库 `.git/objects` 等内部写入不会引发刷新风暴。
- 单仓库项目 UI 保持不变；主仓库变更列表不再展示嵌套子仓库目录条目。
- 文件 Diff 弹窗支持按 Esc 键关闭。

## 0.2 V1.2.4 增补

### 0.2.1 左侧项目列表与终端粘贴

- 左侧项目列表显示每个项目当前已打开的终端数量。
- 开启“按项目聚焦终端”后，点击项目时右侧终端区域只展示该项目关联的终端；点击“全部终端”恢复完整终端列表。
- 终端所有粘贴入口都会先去除剪贴板内容首尾换行。
- 当 `Ctrl+Shift+V` 粘贴内容仍包含换行时，会自动加单引号包裹，避免多行文本被逐行立即发送。
- 普通 `Ctrl+V` 和右键菜单粘贴只做首尾换行清理，不额外包裹内容。

### 0.2.2 推荐操作流程

1. 在左侧项目列表中通过搜索定位项目。
2. 单击项目，右侧终端区域只显示该项目已打开的终端。
3. 需要查看全部终端时，点击项目列表顶部“全部终端”；如不需要该行为，可在设置-终端中关闭“按项目聚焦终端”。
4. 需要安全粘贴多行提示词时，在终端中使用 `Ctrl+Shift+V`。

### 0.2.3 macOS 内置终端环境

- macOS 上选择内置 `zsh` 时，会按 login shell 启动并加载 `~/.zprofile` 等登录环境配置。
- macOS 上选择内置 `bash` 时，会按 `--login -i` 启动并加载 `~/.bash_profile` 等登录环境配置。

## 0.1 V1.2.3 增补

### 0.1.1 左侧项目列表与终端聚焦

- 项目列表支持瞬时搜索：当左侧项目树有焦点时，可直接输入英文开始搜索，也支持在项目列表内按 `Ctrl+F` 呼起搜索框。

- 开启“按项目聚焦终端”后，左侧项目列表顶部会出现“全部终端”虚拟项；点击项目时，右侧仅展示该项目关联的终端与 sub-agent transcript。

3. 搜索过程中可随时按 `Escape` 或清空关键词退出；点击项目列表空白区域后，仍可继续输入搜索或按 `Escape` 收起搜索框。

## 1. 产品定位

CLI-Manager 是一款面向 Windows 开发者的多项目 CLI 工作台，核心目标是把多项目开发中的 **项目、终端、命令、AI CLI 会话、历史回看、统计与同步** 集中到一个桌面应用中管理。

当前项目已经覆盖：

- 多项目与分组管理
- 内置 PTY 终端
- Tab 与 JetBrains 风格分屏
- Claude Code / Codex CLI Hook 状态桥接
- 命令模板与命令历史
- Claude / Codex 历史会话回看
- Diff 查看
- 历史用量分析看板（ccusage 风格）
- 历史会话统计面板与终端实时统计面板
- cc-switch 供应商集成与项目级供应商切换
- WebDAV 云同步
- 主题、快捷键、终端背景等个性化设置

## 3. 内置终端工作区

### 3.1 PTY 终端能力

- 基于 Rust 后端 PTY 创建真实终端会话
- 前端使用 xterm.js 渲染终端
- 支持终端输入写入 PTY
- 支持终端尺寸 resize 同步
- 支持关闭 PTY 会话
- 支持查询终端进程状态
- 后端输出按安全边界切分，避免截断 UTF-8 字符或 ANSI 转义序列
- PTY resize 有最小尺寸保护，避免不可见容器导致异常尺寸

### 3.2 Tab 管理

- 新建终端 Tab
- 关闭当前 Tab
- 重命名 Tab
- 复制 Tab（按当前 Tab 的项目、路径、Shell、环境变量、启动命令和标题快速创建同配置的新终端）
- Tab 拖拽排序
- Tab 溢出滚动
- Tab 溢出列表控制
- Tab 溢出列表浮层跟随终端菜单配色，避免透明模糊影响可读性
- 关闭其它 Tab
- 关闭左侧 Tab
- 关闭右侧 Tab
- 切换 Tab 时保留终端缓冲区
- 隐藏 Tab 有界输出缓冲，降低后台占用

### 3.3 终端状态展示

- Tab 状态点
- 项目树聚合状态点
- 状态类型：
  - 运行中
  - 待审批
  - 已完成
  - 异常退出
- 状态优先级：待审批 > 异常退出 > 运行中 > 已完成
- 基于 OSC 133 shell integration 的标准化状态检测
- PowerShell / pwsh 使用 history id 判断命令真正执行
- 支持 OSC 133 / 633（VS Code）/ 777（私有）三种序列
- 支持 BEL 与 ST 两种终止符
- 跨 chunk 前缀缓冲
- Git Bash / CMD 注入 133 标记
- Shell runtime monitoring 可在设置页关闭（默认关闭以提升启动速度）

### 3.4 终端输出与输入优化

- 高频 PTY 输出合并后限频写入 xterm
- 非激活终端降频刷新
- WebGL 渲染失败或上下文丢失时回落 Canvas 渲染
- 中文输入法组合输入期间暂停自动 fit，降低界面抖动
- 终端粘贴支持
- 支持从系统文件管理器拖拽文件到终端并显示本地路径
- 终端复制支持
- Codex 高频重绘场景下合并光标显示序列，减少闪烁
- 终端输出搜索浮层
- `Ctrl+F` 搜索终端输出
- 搜索结果高亮
- 上一个 / 下一个搜索结果导航
- 关闭搜索后恢复终端焦点
- OSC 8 超链接通过系统默认浏览器打开
- 终端回滚行数可配置（1000-50000 行，默认 5000）
- 终端回滚行数支持热更新，无需重启应用

### 3.5 终端中文输入法修复

- 修复中文输入法下输入中文符号时需要输入两次的问题
- 修复中文输入法候选框错位问题
- 修复 Claude Code / Codex 流式重绘导致候选框漂移
- IME composition 锚点冻结增强
- IME 候选框锚点增加静默光标采样、TUI 边框输入行识别
- 纯结构识别候选框锚点位置，免疫 TUI 流式重绘
- 支持空框 / 单行 / 多行场景

## 6. Claude Code / Codex CLI 集成

### 6.1 Hook Bridge

- 应用启动本地回环 Hook 通知服务
- 仅监听 `127.0.0.1`
- 使用 bearer token 校验 Hook 请求
- 为每个 PTY 会话注入：
  - `CLI_MANAGER_TAB_ID`
  - `CLI_MANAGER_NOTIFY_PORT`
  - `CLI_MANAGER_NOTIFY_TOKEN`

### 6.2 Claude Code Hook

支持事件：

- `UserPromptSubmit`
- `Notification`（细分 matcher：permission_prompt / idle_prompt 置 attention，auth_success 等不干扰）
- `Stop`
- `StopFailure`

## 20. 安全与边界处理

### 20.1 历史会话安全

- 删除历史会话前校验来源
- 删除历史会话前校验项目边界
- 删除历史会话前 canonicalize 路径
- 只允许删除受支持来源下的 `.jsonl` 文件
- 切换 Claude / Codex 配置目录后隔离历史索引与缓存

### 20.2 Hook Bridge 安全

- Hook 服务仅监听本地回环地址
- Hook 请求必须携带 bearer token
- Hook body 有大小限制
- Hook header 有大小限制
- 只接受指定事件类型

### 20.3 背景图片安全

- 图片来源必须是绝对路径
- 只允许 `jpg` / `jpeg` / `png` / `gif`
- 禁止 `webp`
- 保存后的相对路径禁止：
  - 空路径
  - `..`
  - 反斜杠
  - 绝对路径
  - 逃逸 `backgrounds/` 目录
- asset protocol scope 锁定到背景图目录

### 20.4 WebDAV 安全与稳定性

- 使用 Basic Auth
- WebDAV 响应大小限制
- 请求超时限制
- 上传前确保远端目录存在
- 旧路径兼容但新数据使用设备快照路径

### 20.5 PTY 稳定性

- PTY 输出避免切断 UTF-8 字符
- PTY 输出避免切断 ANSI / OSC / CSI 转义序列
- 未完成残尾延迟到下一轮拼接
- 异常源端有缓冲兜底，避免无限增长
- resize 有最小列数 / 行数保护

## 23. 简短结论

CLI-Manager 当前的基础能力已经比较完整，主要价值集中在三件事：

1. **把多项目终端集中起来**：项目树、内置终端、分屏、外部终端。
2. **把 Claude / Codex 使用过程管理起来**：Hook 状态、通知、历史会话、Diff、Prompt。
3. **把长期使用数据沉淀下来**：命令模板、命令历史、统计看板、WebDAV 同步。

后续最自然的增量方向不是继续堆普通终端功能，而是围绕 README Roadmap 做：

- Claude 配置同步增强
- 项目启动 Profile / Run Profile
